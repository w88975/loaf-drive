# Change: 预览弹框响应式设计

## Why
当前的文件预览弹框（PreviewModal）在所有设备上使用统一的样式：固定最大宽度（max-w-5xl）、居中显示、带外边距和阴影。这在移动端造成以下问题：
1. 可视区域受限：移动端屏幕本就较小，弹框边距进一步压缩了预览内容的显示空间
2. 用户体验不佳：移动端用户期望全屏沉浸式的预览体验，而非缩小的桌面端样式
3. 交互不便：小屏幕下按钮和文字过于密集，操作体验差

通过实现响应式设计，可以在保持桌面端体验的同时，为移动端用户提供更适配的全屏预览界面。

## What Changes
- **PC端（≥768px）**：保持当前样式不变（居中弹框、max-w-5xl、padding、阴影）
- **移动端（<768px）**：改为全屏显示
  - 移除外层 padding（`p-4` → `max-md:p-0`）
  - 移除最大宽度限制（使用 `max-md:max-w-none` 或条件类）
  - 调整高度为 100vh（`max-md:h-screen`）
  - 移除或调整阴影效果（移动端全屏不需要阴影）
  - 保持所有功能按钮和交互不变

## Impact
- Affected specs: `file-preview`（修改现有 spec，添加响应式预览需求）
- Affected code:
  - `components/overlays/PreviewModal.tsx` - 添加响应式 Tailwind 类
- 用户体验：
  - 移动端用户获得全屏沉浸式预览体验
  - PC端用户体验保持不变
  - 无破坏性变更，纯 UI 调整
- 性能：无影响（仅 CSS 类变化）
